# MacroNode Editor (시각적 매크로 자동화 툴) 완벽 가이드

이 프로그램은 코딩 없이 **노드(Node)를 선으로 연결하여** 누구나 쉽게 화면 인식 기반의 자동화 매크로를 만들 수 있는 시각적 에디터입니다.
"현재 화면에 A가 보이면 B를 누르고 변수를 C만큼 증가시킨다"는 규칙(Rule)들을 엮어 스스로 판단하는 매크로를 작성할 수 있습니다.

---

## 📂 1. 전체 구조 (Architecture)

프로젝트는 크게 3가지 영역으로 나뉩니다.

*   **`ui/` (시각적 에디터 UI 화면)**
    *   `node_editor_main.py`: 에디터를 실행하는 진입점입니다.
    *   `custom_nodes.py`: 에디터상에 표시되는 각 노드(Start, Plan, Guard, Rule)들의 속성 정의입니다.
    *   `panels/`: 우측의 상세 속성 조작 패널(`rule_panel.py`, `plan_guard_panel.py` 등)들입니다.
*   **`framework/` (엔진 코어)**
    *   `flow_runner.py`: **[핵심]** 사용자가 연결해둔 그래프를 해석하여 실제로 마우스를 움직이고 이미지를 매칭하며, 여러 분기와 변수 조건들을 판단하는 무한 루프 뇌 역할을 합니다.
*   **`library/` (하드웨어/기기 제어)**
    *   `macro_manager.py`: 화면 캡처, 이미지 찾기, 클릭, 드래그 등의 동작 함수입니다.

## 🚀 2. 노드의 종류와 역할

직접 코드를 짤 필요 없이, 에디터 화면에서 노드를 생성하고 서로 연결(Link)(`in`, `out` 포트 드래그)하기만 하면 됩니다.

### 1) [Start] 노드 (시작점)
*   **색상:** 출력(out) 포트가 **초록색**으로 표시됩니다. 노드 본체는 기본 어두운 색상입니다.
*   **역할:** 모든 매크로의 기준점. 1개만 존재합니다.
*   **초기 설정:** 속성창에서 연결할 기기(ADB)를 설정합니다.
*   **⏰ 일일 초기화 시간:** 하루의 시점(예: 오전 5시 30분)이 언제부터 시작되는지 타임라인을 지정하여, 변수 기록 파일(`YYYYMMDD.txt`)의 생성 일자 기준을 정합니다.
*   **변수 선언:** `Daily Variables` 속성에서 오늘 하루 동안 누적 기록하고 활용할 전역 **변수(Variable)** 들(예: `GOLD`, `던전입장권` 등)을 선언할 수 있습니다. 이 변수들은 `daily_records/` 폴더에 파일 형태로 영구 보존됩니다. 
    *   *(공백 제출을 차단하는 안정성 검증이 포함되어 있습니다.)*

### 2) [Plan] 노드 (작업 그룹화)
*   **색상:** 입력(in) 및 출력(out) 포트가 **하늘색(파란색 계열)**으로 표시됩니다.
*   **역할:** 특정 작업의 큰 단위(예: "일일 퀘스트 플랜", "우편함 확인")를 하나로 묶어 관리합니다. 자체적인 내부 카운터 루프 대신, Rule 노드들의 변수 연산 및 실행 제한 조건을 통해 유연하게 반복 연산을 제어합니다.
*   **⏰ 개별 초기화 시간 사용:** 필요에 따라 특정 Plan만 Start 노드의 전역 시간 기준을 따르지 않고, 개별적으로 일일 초기화 넘버링 타임을 덮어씌워서 실행되도록 설정할 수 있습니다.
*   **완료 카운트 변수명(선택):** 비워두어도 자동으로 집계됩니다. 저장 파일에서는 Game/Plan 구조로 정리되어 표시됩니다.

### 3) [Guard] 노드 (돌발 상황 감시)
*   **색상:** 노드 본체와 포트가 모두 **주황색**으로 표시되어 시각적으로 강하게 구분됩니다.
*   **역할:** 매크로가 도는 내내 예기치 않은 팝업창이나 에러 문구가 뜨는지 최우선으로 감시합니다. Guard를 타면 돌발 상황을 해결하고 다시 원래의 Plan으로 복귀하게 설계할 수 있습니다.

### 4) [Rule] 노드 (핵심 요소)
*   **색상:** 입력(in) 포트는 **빨간색**, 출력(out) 포트는 **초록색**으로 표시됩니다.
*   실질적인 화면 인식과 행위를 정의하는 행동 대장입니다.
*   **🔍 조건 (Conditions):** 화면에서 찾을 이미지나 픽셀 조건. 여러 개 등록 가능(모두 만족해야 matched).
    *   *(캡처된 이미지는 파일이 아닌 **Base64 텍스트** 형태로 노드 내부에 직접 보존되어 배포 및 관리 편의성이 높습니다.)*
*   **🖱️ 동작 (Actions):** 조건 통과 시 수행할 클릭, 대기, **🧮 변수 변경** 등을 등록합니다.
    *   *동작 리스트에 변수 연산을 포함시켜 마우스 클릭 등과 상호 유기적인 흐름 설계가 용이합니다.*
*   **🔀 분기 순서 (Out Nodes Order):** 이 노드가 성공했을 때 이동할 다음 노드가 여러 개일 경우 검사 순서를 우선순위 목록으로 지정합니다.
*   **⏱ 다음 Rule 탐색 제한(초):** 다음 Rule이 일정 시간 안에 매칭되지 않으면 직전 Rule로 복귀합니다. Rule마다 개별 설정 가능하며 기본값은 5초입니다.

## 🖱️ 3. 노드 편집 조작법

에디터 중앙의 노드 그래프 화면에서 노드를 추가, 삭제, 이동할 수 있습니다.

*   **노드 추가:** 빈 공간에서 마우스 오른쪽 버튼을 누른 뒤 `Add Node` 메뉴에서 Start, Game, Plan, Rule, Guard 노드를 선택합니다.
*   **노드 실행 시작점 지정:** 노드 위에서 마우스 오른쪽 버튼을 누른 뒤 `▶ 여기서부터 실행`을 선택하면 해당 노드부터 매크로를 시작합니다.
*   **노드 1개 삭제:** 노드 위에서 마우스 오른쪽 버튼을 누른 뒤 `이 노드를 삭제`를 선택하면 해당 노드만 삭제합니다.
*   **하위 노드까지 삭제:** 노드 위에서 마우스 오른쪽 버튼을 누른 뒤 `이 노드 하위도 같이 삭제`를 선택하면 해당 노드의 `out` 연결을 따라 이어진 하위 노드들도 함께 삭제합니다.
*   **하위 노드 묶음 이동:** 노드 위에서 마우스 휠 버튼을 누른 채 드래그하면 해당 노드와 `out` 방향 하위 노드들이 같이 이동합니다.
*   **화면 이동:** 빈 공간에서 마우스 휠 버튼을 누른 채 드래그하면 노드 그래프 화면을 이동합니다.
*   **확대/축소:** 마우스 휠을 위아래로 굴려 노드 그래프를 확대하거나 축소합니다.
*   **선택 노드 삭제:** 노드를 선택한 뒤 `Delete` 키를 누르면 선택된 노드를 삭제합니다.

삭제 작업은 확인창을 거친 뒤 실행되며, 매크로 실행 중에는 노드 삭제가 제한됩니다.

## 💡 4. 고급 기능: 변수 활용과 실행 제한 조건

이 매크로 엔진의 가장 강력한 기능은 **변수를 조작하고 그에 따라 실행을 동적으로 제어**하는 기능입니다.

### 일일 변수(Daily Variables)와 기록의 영속성
매일매일 실행 상태를 기억합니다. 사용자가 설정한 **일일 초기화 시간**을 기준으로 계산된 `daily_records/output_YYYYMMDD.txt` 파일에 사용자가 조작한 **전역 변수(Variable)의 수치만 한 줄씩 텍스트 형태로 적재**됩니다. 엔진 자체의 노드 카운트는 더 이상 내부 기록되지 않으므로 직관적입니다. 
- ⏱️ 또한 실행 시 콘솔 창에 `[HH:MM:SS]` 형태의 타임스탬프와 함께 변수 갱신 내역 및 판단 내역이 투명하게 출력되어 매크로의 흐름을 정확히 짚고 넘길 수 있습니다.
- Plan 완료 카운트는 **정상 완료(마지막 Rule 이후 다음 Rule 없음)** 에서만 증가하며, 가드 점프/재시작/제한 조건 중단 경로에서는 증가하지 않습니다.
- Guard 카운트는 **Guard 발동 시점**(돌발 상황 감지 후 Guard 흐름 진입)마다 증가합니다.
- 기록 파일은 아래 형태로 출력됩니다.
  - `[게임 라벨]`
  - ` - [플랜]`
  - `  - 플랜 라벨: N`
  - ` - [가드]`
  - `  - 가드 라벨: K`
  - ` - [변수]`
  - `  - 변수 라벨: M`

### 🛑 실행 제한 조건 (Execution Restriction Conditions)
`Plan`, `Guard`, `Rule` 노드 패널에는 **실행 제한 조건(Break Conditions)** 편집 기능이 존재합니다.
*   **역할:** "만약 선언했던 `포션` 변수가 `0` 이하라면, 이 노드를 아예 **실행 불가 상태로 만들고 다른 경로를 찾거나 이 플랜 루프를 즉시 끊음(Break)**"
*   단순 횟수 카운팅을 너머, 게임 내의 자원이나 아이템 변동량에 맞춰 지능적으로 스탑 또는 분기 우회 할 수 있습니다.

## 🛠 5. 실행 및 배포 방법

1.  **에디터 실행:** `python ui/node_editor_main.py`
2.  노드 구성 후 캡처/설정을 마치고 `Save Flow` 버튼을 눌러 `.json` 로 현재 맵 뼈대를 저장합니다.
3.  **▶ 메인 매크로 시작:** (프로젝트의 매크로 스타트 진입점을 연동하거나 호출하여 사용합니다).

배포 시엔 작성한 매크로 파일(`.json`)만 포함시켜주면 다른 환경에서도 똑같이 구동됩니다! (**Base64** 방식으로 캡처 이미지가 파일 내부에 직접 포함되므로 별도의 `captures/` 폴더가 필요하지 않습니다.)
