## 1) 처음 사용자 빠른 시작

### ZIP 실행

배포된 ZIP 파일을 압축 해제한 뒤 아래 실행 파일을 실행하세요.

```text
SheerHeartAttack.exe
```

실행 후 상단 ADB 툴바에서 포트를 입력하고 연결하면 편집/실행 기능이 활성화됩니다.

---

## 2) ADB 연결 (필수)

툴 사용 전 ADB 연결이 반드시 필요합니다.

### 블루스택에서 포트 확인

1. 블루스택 실행
2. 설정
3. 고급 기능 설정
4. Android 디버그 브리지 (ADB)

표시 예시:

```text
127.0.0.1:5555
```

툴의 **ADB 포트** 입력란에 `5555`를 입력하고 **연결** 버튼을 누르세요.

---

## 3) 매크로 파일 저장/불러오기

- 기본 저장 폴더: `macros/`
- 파일 형식: `.json`
- 권장 순서:
  1. 새 매크로 생성
  2. 노드 구성 및 설정
  3. 저장 (`Ctrl+Alt+S` 또는 `Ctrl+Shift+S`)
  4. 다음 실행부터 불러오기 (`Ctrl+O`)

---

## 4) 노드 시스템

시어하트 어택은 노드 기반으로 작동합니다.

### Start

매크로의 시작점입니다.

- 반드시 필요합니다.
- Start가 없으면 매크로가 실행되지 않습니다.
- 현재 여러 개가 생성되지만 **하나만 사용해주세요.**

### Game

실행할 게임 단위를 설정하는 노드입니다.

- 현재 실행 중인 앱 패키지 감지
- 앱 패키지 실행 가능
- `앱 실행 후 대기(초)` 설정 가능

연결 구조:

```text
Start → Game
```

### Plan

게임 루틴을 묶는 노드입니다.

- 작업 흐름 관리
- 반복 루틴 구성

연결 구조:

```text
Game → Plan
```

### Rule

조건과 동작을 설정하는 핵심 노드입니다.

- 조건 확인
- 동작 실행
- 다음 Rule 탐색 제한 시간 설정 가능

연결 구조:

```text
Plan → Rule
```

### Guard

예외 대응용 보조 루틴 노드입니다.

- Game/Plan 단위에서 예외 상황 대응
- 팝업, 경고창 등 돌발 상황 처리

연결 가능:

```text
Game → Guard
Plan → Guard
```

### 실행 가능한 노드 조건

아래처럼 연결된 노드만 실행됩니다.

```text
Start → Game → Plan → Rule
```

연결되지 않은 노드는 무시됩니다.

## 5) 노드 생성/편집

- 맵 빈 공간 **우클릭**: 노드 생성
- 노드 **우클릭**: 여기서부터 실행, 노드 삭제, 하위 포함 삭제
- `Delete`: 선택 노드 삭제
- 실행 중에는 맵 편집이 제한됩니다.

---

## 6) Rule 설정

Rule에서는 조건과 동작을 설정할 수 있습니다.

### 화면 캡처

캡처 버튼을 누르면 현재 화면을 등록하여 조건 확인에 사용합니다.

### 조건 추가

1. 조건 탭 클릭
2. 조건 추가 클릭
3. 조건 타입 선택
4. 세부 설정

### 동작 추가

1. 동작 탭 클릭
2. 동작 추가 클릭
3. 동작 타입 선택
4. 세부 설정

주의:

- 현재 조건 탭과 동작 탭의 선택 상태가 구분이 안됩니다.
- 반드시 탭을 먼저 선택한 뒤 추가 버튼을 누르세요.

---

## 7) 실행 순서와 반복

### 다중 연결 실행 순서

하나의 노드에 여러 노드가 연결되면 실행 순서를 설정할 수 있습니다.

설정 가능한 노드:

- Start
- Game
- Plan
- Rule

### Guard가 있는 경우 실행 흐름

```text
Game Guard
↓
Game Plan
↓
Plan Guard
↓
Plan Rule
```

### 전체 게임 루틴 반복

Start에 Game을 여러 개 연결하면 순서대로 반복됩니다.

```text
Start
 ├─ Game1
 ├─ Game2
 └─ Game3
```

다음 Game으로 이동하는 시점:

- 현재 Game에서 실행 가능한 Plan/Guard가 모두 없을 때

## 8) 기록 파일 (`daily_records`)

Plan의 마지막 Rule이 정상 완료되면 기록이 저장됩니다.

- 생성 폴더: `daily_records/`
- 파일명 형식: `output_YYYYMMDD.txt`
- 기록 기준 날짜는 Game/Plan의 초기화 시간 설정을 반영합니다.

기록 예시 항목:

- Plan 완료 카운트
- Guard 실행 카운트
- 변수 값

---

## 9) 실행 방법

- 실행: `F5`
- 정지: `F6`

실행 전 확인:

- ADB 연결 완료
- 노드 연결 완료
- Rule 조건/동작 설정 완료
- 변경사항 적용 (`Ctrl+S`)
